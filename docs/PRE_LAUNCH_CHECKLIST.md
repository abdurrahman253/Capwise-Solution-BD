# Capwise Pre-Launch Checklist

## Required technical checks

- `npm ci`, `npm run lint`, `npm run build` and `npm run audit:routes` pass.
- `/api/health` reports reachable MongoDB and configured email.
- A real consultation reaches the team mailbox with working Reply-To.
- Failed email delivery still leaves a traceable MongoDB consultation record.
- Vercel Production and Preview environment variables are separately checked.
- Domain, HTTPS, canonical URLs, sitemap, robots and Open Graph image work.
- Keyboard navigation, reduced motion, light/dark mode and small-phone layouts are tested.

## Required client approvals

- Team photos, credentials and final bios
- Case studies, testimonials and logo permission
- Guide PDFs and source review
- Official Facebook, LinkedIn, WhatsApp and Google Maps details
- Legal review of Privacy, Terms and Professional Disclaimer
- Data retention, deletion, analytics and cookie decisions
- Production sender domain, SPF, DKIM and DMARC

## High-value operational upgrades after launch

- Authenticated lead-management portal with owner, status, next action and response target
- Secure document-request workflow instead of public uploads
- Reviewed regulatory publishing workflow with named reviewer and review-due date
- Search Console, Bing Webmaster and privacy-approved conversion measurement
- Quarterly accessibility, performance, security, route and regulatory-content review
